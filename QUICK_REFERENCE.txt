ASSESSMENT DIMENSIONS
─────────────────────────────────────────────────────────────────────────────────

HYBRID FRAMEWORK: 9 Dimensions, 0-100 Scoring

LAYER 1: CORE QUALITY (6 dimensions × 50% weight)
  1. Commands/Workflows → Actionable implementation guidance?
  2. Architecture Clarity → Can engineers understand the context?
  3. Non-obvious Patterns → Are gotchas and edge cases documented?
  4. Conciseness → Scannable, no fluff?
  5. Currency → Tools/threats current (2025-2026)?
  6. Actionability → Instructions executable, not vague?

LAYER 2: SECURITY-SPECIFIC (3 dimensions × 50% weight)
  7. Threat Coverage → Addresses current attacks? Maps to OWASP/CWE?
  8. Compliance & Standards → Aligns with NIST, ISO27001, regulations?
  9. Cross-Skill Integration → Clear dependencies? No duplication?

Composite Score = (Layer 1 avg × 50%) + (Layer 2 avg × 50%)
  • A (90-100) = Comprehensive, current, actionable ✅
  • B (80-89) = Good coverage, minor gaps ✅
  • C (70-79) = Basic coverage, some gaps ⚠️
  • D (60-69) = Sparse/outdated ⚠️
  • E (50-59) = Significant gaps ❌
  • F (<50) = Critical gaps ❌

TROUBLESHOOTING
─────────────────────────────────────────────────────────────────────────────────

PROBLEM: Script won't run
SOLUTION: PowerShell execution policy
  Set-ExecutionPolicy -ExecutionPolicy Bypass -Scope Process

PROBLEM: "File not found" error
SOLUTION: Ensure prompt file and script in same folder as skills directory

PROBLEM: File too large for Claude
SOLUTION: Assess by domain instead
  Example: .\prepare-skills-assessment.ps1 -SkillsDir "skills\backend"

PROBLEM: Claude output cuts off
SOLUTION: Ask Claude to continue:
  "The response was cut off. Please continue the report from where you left off."

PROBLEM: Unclear what gaps mean
SOLUTION: See SETUP_AND_USAGE_GUIDE.md for detailed explanations
